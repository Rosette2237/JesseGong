Current project

GT Movie Store Description

  1. The app provides an informational Home and About section using Django templates rendered from the home application. These pages are served through the standard URL routing, view functions, and HTML templates that extend a shared base layout.
  2. User registration is implemented with Django’s built-in authentication system by providing a signup page that creates a new User record. After successful registration, the user is redirected so they can begin using protected features of the store.
  3. Login is implemented through Django authentication so a registered user can securely access their account session. Once authenticated, Django maintains the login state using sessions, allowing the user to navigate the site without re-entering credentials on each page.
  4. The movies list is implemented with a Movie database model and a movies index page that queries the database and displays the results. The output is rendered in a template using Bootstrap styling to show each movie in a consistent layout.
  5. Movie search is implemented by reading a query string from the URL and filtering Movie objects with a case-insensitive title match. The filtered results are passed back to the same list template so the user can quickly refine selections.
  6. The shopping cart is implemented using Django sessions so cart data persists across page navigations without requiring an account-based cart model. A dedicated cart page reads the session data and renders the current cart items for the user.
  7. Adding items to the cart is implemented with a POST form on the movie detail page that submits the desired quantity. The cart view updates the session-stored cart dictionary so the user can accumulate items over time.
  8. Review creation is implemented with a Review model connected to both the Movie and the authenticated User. A form on the movie detail page sends review content to a create-review view that saves it and refreshes the page to show the new entry.
  9. Clearing the cart is implemented as a dedicated action that removes the cart data from the user’s session storage. After the session is updated, the user is redirected back to the cart page where the empty state is displayed.
  10. Editing reviews is implemented by providing an edit view and template that loads the existing Review and saves the updated content on submission. Authorization logic ensures only the owner of a review is allowed to access the edit functionality.
  11. Deleting reviews is implemented through a delete view that removes the selected Review from the database and then redirects back to the related movie page. Ownership checks prevent users from deleting reviews they did not create.
  12. Viewing reviews for a movie is implemented by querying all Review objects linked to the selected Movie and sending them to the movie detail template. The template renders the review list in the comment section so users can evaluate feedback before purchasing.
  13. Viewing movie details is implemented using a dedicated movie detail page that loads one Movie object and displays its fields like title, description, price, and image. The template uses responsive styling so images and content stay within the page layout across screen sizes.
  14. The orders list is implemented using Order records associated with the logged-in user and displayed through an orders page. The page queries the user’s order history and renders it in a readable format so the user can track purchases and spending.
  15. Access from any desktop browser is achieved by building the application as a standard Django web platform that runs on a web server and is reachable through a URL. The deployment setup ensures the site can be accessed over the internet with normal browser navigation.
  16. A responsive GUI is achieved by using Bootstrap components such as a responsive navbar, grid layout, and utility classes. The templates and CSS are designed so content scales and rearranges appropriately for different screen sizes.
  17. Administrator management of users is provided through Django’s built-in admin panel, which includes full user CRUD operations for authorized staff. Admin accounts can view, create, update, or remove users to keep account information current.
  18. Administrator management of movies is implemented by registering the Movie model in the Django admin and configuring ModelAdmin options such as searching and ordering. This allows admins to add new movies, edit existing data, or delete entries quickly.
  19. Administrator management of reviews is implemented by registering the Review model in the admin and enabling filters and searches for moderation. Admins can update or remove reviews to maintain quality and enforce content standards.
  20. Administrator management of orders is implemented by registering Order and related item models in the admin interface. This allows staff to inspect transactions and maintain order records in an organized way.
  21. Reporting inappropriate reviews is implemented by adding a report action on each review and handling it with a protected view that records the report and hides the review. Hidden reviews are excluded from future page rendering so the comment section stays relevant and clean.

Process description

I worked on this project by following the textbook’s structure and progression, and I used an agile approach: I started directly from the user stories, implementing one feature at a time until each story’s behavior was working end-to-end.

As I implemented each feature, I went back into the textbook to find the corresponding sections that matched what I needed (URLs , views, templates, then models/migrations, auth, sessions, admin, etc.).

Whenever I got stuck or hit unexpected errors, I used Stack Overflow and online searches/documentation to troubleshoot and resolve the issue efficiently.
